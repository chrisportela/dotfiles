# CLAUDE.md

Guidance for AI assistants working in this repository.

## Project overview

Bare React Native app with TypeScript, pnpm, and Nix. Uses React Navigation (bottom tabs + native stack). Native projects (`android/`, `ios/`) are generated by `./setup.sh MyApp`; the template ships only the JS/TS source and config.

## Commands

```bash
pnpm start     # Metro bundler
pnpm android   # Run on Android
pnpm ios       # Run on iOS (macOS)
pnpm test      # Jest
pnpm lint      # ESLint
./setup.sh MyApp   # Generate android/ and ios/ (one-time)
```

## Layout

- **Entry:** `index.js` registers the app from `app.json` and mounts `App`.
- **App:** `App.tsx` wraps the app in `NavigationContainer` and `RootNavigator`.
- **Navigation:** `src/navigation/RootNavigator.tsx` — bottom tabs; Home tab is a stack (Home → Details), Settings is a single screen.
- **Screens:** `src/screens/` — HomeScreen, DetailsScreen, SettingsScreen.
- **Path alias:** `@/*` → project root (see `tsconfig.json` and `jest.config.js`).

## Nix

- `flake.nix` provides a single output: `devShells.default`.
- Shell includes Node, pnpm, Android SDK (writable copy in `.android-sdk/`), JDK, Gradle; on Darwin: CocoaPods and Xcode.
- No Nix production build; APK/IPA are built with Gradle/Xcode.

## Testing

- Jest with `react-native` preset; `moduleNameMapper` for `@/`.
- Tests live in `__tests__/`; use `@testing-library/react-native` for component tests.

## Key files

- `app.json` — `name` must match the component name expected by native (set by `setup.sh`).
- `metro.config.js` — Metro config; `babel.config.js` uses `@react-native/babel-preset`.
- `.npmrc` — `node-linker=hoisted` is required for React Native with pnpm.
